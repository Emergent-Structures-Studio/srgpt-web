# Biz/SI と Codex の接続（初稿）

この文書は、Biz/SI と Codex を「実装前構造を整える」文脈で接続する最小ガイドです。

## Biz/SIがCodexに渡す前に整える5条件

- なぜ変えるのか
- 何を変えるのか
- どこまで変えるのか
- 何を壊してはいけないのか
- 何をもって完了か

## 接続フレーム

- why change（なぜ変えるか）: 何の混乱・重複・手戻りを減らすのか
- what to change（何を変えるか）: 文書、構成、静的ページなど対象を限定する
- how far to change（どこまで変えるか）: 初稿で止める範囲と次段を分ける
- what must not be broken（壊してはいけないもの）: 既存の公開説明との整合
- what counts as done（完了の定義）: 読める、レビューしやすい、矛盾しない

## Codex の支援範囲（最小）

- Markdown 文書の下書き
- ファイル構成の追加・整理
- 小さなPR説明文の作成補助

## 注意

内部プロンプト・非公開仕様・未公開OS詳細は、公開文書には含めません。
