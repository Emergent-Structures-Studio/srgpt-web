# SRGPTシリーズ・マップ（公開説明用）

この文書は、SRGPTシリーズを「用途別GPT一覧」ではなく、**観測軸から選ぶための入口**としてまとめた最小マップです。

## 使い方（最短）

1. いまの相談を、先に「何を観測したいか」で見る
2. 近い観測軸のシリーズを1つ選ぶ
3. 入口文をそのまま使って相談を始める

---

## SRGPT Core 7.5
- **観測軸**: 問いの軸（問いの輪郭・前提・論点の置き方）
- **ひとこと説明**: テーマを急いで結論化せず、まず問いの形を整える基盤。
- **向いている相談**: 何から考えるべきか曖昧な相談、論点が散らばる相談。
- **入口文**: 「このテーマを、まず問いの形から整理してください。」

## SRGPT Core 7.5L
- **観測軸**: 問いの軸 + 身体の軸（負荷・継続可能性・生活実感）
- **ひとこと説明**: 考える内容と続けられる運用感を同時に観測する軽量系。
- **向いている相談**: 考えたいが疲弊しやすい相談、長く続ける前提の相談。
- **入口文**: 「問いの整理に加えて、無理なく続けられる形で見てください。」

## SRGPT Lab+ 7.5
- **観測軸**: 問いの軸 + 検証の軸（仮説・試行・学習ループ）
- **ひとこと説明**: 未確定なテーマを、試せる仮説に分解して前進させる探索系。
- **向いている相談**: 正解が固定されていない企画、試行しながら精度を上げる相談。
- **入口文**: 「このテーマを、検証可能な仮説に分けて進め方を作ってください。」

## SRGPT Biz/SI 7.5
- **観測軸**: 実装前構造の軸（目的・範囲・制約・完了条件の先行定義）
- **ひとこと説明**: 実装前に構造を先に整え、手戻りを減らして合意可能性を上げる。
- **向いている相談**: 要件整理、業務設計、導入前の役割分担や判断基準の明確化。
- **入口文**: 「実装前提で、目的・範囲・制約・完了条件を先に構造化してください。」

## SRGPT Spo/Golf 7.5
- **観測軸**: 身体の軸 + 関係の軸（感覚・再現性・伴走関係）
- **ひとこと説明**: 技術論だけでなく、身体感覚と対話を接続して再現性を高める。
- **向いている相談**: スイング改善、練習設計、指導・自己観察の言語化。
- **入口文**: 「身体感覚と言語をつなぎ、再現できる練習の形にしてください。」

---

## 補足

- SRGPTは、シリーズごとに観測軸が異なります。
- 同じテーマでも、どの軸で観測するかで進め方が変わります。
- 迷う場合は Core 7.5 から入り、必要に応じて他シリーズへ移動してください。
