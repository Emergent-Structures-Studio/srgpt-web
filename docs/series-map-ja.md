# SRGPTシリーズ・マップ（公開説明用）

この文書は、SRGPTシリーズを「用途別GPT一覧」ではなく、**観測軸から選ぶための入口**としてまとめた最小マップです。

## 使い方（最短）

1. いまの相談を、先に「何を観測したいか」で見る
2. 近い観測軸のシリーズを1つ選ぶ
3. 入口文をそのまま使って相談を始める

---

## SRGPT Core 7.5
- **観測軸**: 生成観測
- **ひとこと説明**: 問いがまだ固まっていない段階で、思考・感情・意味・問い・違和感の立ち上がりを見る入口OS。
- **向いている相談**: 何を問うべきか曖昧な相談、言語化前の引っかかりがある相談。
- **入口文**: 「まだ問いが固まっていないので、生成観測として、引っかかりと背景構造から整理してください。」

## SRGPT Core 7.5L
- **観測軸**: 変調観測
- **ひとこと説明**: 応答変調観測レンズとして、GPT応答・対話の揺れ・認知の変調・違和感・レンズ感度を見るOS。
- **向いている相談**: 答えの正否だけでなく、応答のズレ方や揺れ方を観測したい相談。
- **入口文**: 「このテーマを、変調観測として見てください。応答の温度差、言葉の選び方、ズレ方から整理してください。」

## SRGPT Lab+ 7.5
- **観測軸**: 変化観測
- **ひとこと説明**: Self-Research × Relation の観点から、自己変化・反復パターン・関係内の学び・成長の兆候・継続ログを見るOS。
- **向いている相談**: 自分や関係の中で起きる変化を継続的に観測し、学びを積み上げたい相談。
- **入口文**: 「Self-Research × Relation の観点で、変化観測として見てください。自己変化、反復パターン、関係内の学び、成長の兆候、継続ログを整理してください。」

## SRGPT Biz/SI 7.5
- **観測軸**: 構造観測
- **ひとこと説明**: 実務・案件・制度・責任・リスク・判断順序を扱い、実装前に構造をそろえるOS。
- **向いている相談**: 要件や責任分界を整理し、判断順序をそろえて実装前の手戻りを減らしたい相談。
- **入口文**: 「この案件を構造観測として整理してください。目的・範囲・制約・完了条件と、要件・責任分界・依存関係・工程・受入条件を先にそろえてください。」
- **Codex接続（短く）**: 実装前に、目的・範囲・制約・完了条件を整える準備層として接続します。

## SRGPT Spo/Golf 7.5
- **観測軸**: 身体観測
- **ひとこと説明**: 身体感覚・動作・再現性・練習ログ・違和感・次回練習テーマを扱うOS。
- **向いている相談**: 練習中の感覚差を言語化し、再現可能な次の練習テーマにつなげたい相談。
- **入口文**: 「この練習を身体観測として整理してください。身体感覚・動作・再現性・練習ログ・違和感から、次回練習テーマまで短くまとめてください。」

---

## 補足

- SRGPTは、シリーズごとに観測軸が異なります。
- 同じテーマでも、どの軸で観測するかで進め方が変わります。
- 迷う場合は Core 7.5 から入り、必要に応じて他シリーズへ移動してください。
