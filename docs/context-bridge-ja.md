# コンテキスト・ブリッジ（公開説明）

この文書は、SRGPTにおける「コンテキスト」を、公開説明向けに最小で整理したものです。

## コンテキスト＝背景構造

SRGPTでいうコンテキストは、単なる前提情報ではありません。
問いや相談が生まれている**背景構造**そのものを指します。

背景構造には、主に次が含まれます。

- 関係（誰と誰がどう関わるか）
- 時間（いつまでに、どの順序で進むか）
- 制約（予算・体制・運用上の限界）
- 判断（どこで何を決めるか）
- 違和感（言語化前の引っかかり）
- 未確定要素（まだ決め切れない論点）

## すぐ固定しない観測

SRGPTは、非構造化された相談やメモを、すぐに固定しません。
まず背景構造として観測し、どこが動く要素で、どこが固定条件かを見分けます。

## 次のOSやCodexへ渡すための圧縮

必要に応じて、背景構造を次のOSやCodexへ渡せる形に圧縮します。
ただし、圧縮後も完全に凍結せず、再観測して更新できる余地を残します。

## Codex接続での圧縮例

Codexへつなぐときは、背景構造を次の5点に圧縮する形が使えます。

- なぜ変えるのか
- 何を変えるのか
- どこまで変えるのか
- 何を壊してはいけないのか
- 何をもって完了か

この5点があると、実装前の意図と境界を保ったまま、レビュー可能な変更に落とし込みやすくなります。
